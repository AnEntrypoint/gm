code_index.rs history (rs-plugkit): project_db_filename/project_db_path -- the libsql plugin became a stateless process-wide instance shared across every concurrently active project, so a bare relative filename like "gm.db" is unsafe (two different projects could collide if the plugin resolved paths against its own process cwd). project_db_path resolves an absolute host_cwd/.gm/filename path fresh every call via host_cwd_string(); project_path=None resolves against the current dispatch's own project root, Some(p) namespaces an external project db by crc32-hash filename but still resolves the directory against current host_cwd.
