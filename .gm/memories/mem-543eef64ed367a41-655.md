A subagent dispatched to build/commit/push in a gm submodule (agentplug, rs-plugkit, rs-codeinsight, agentplug-bert, agentplug-libsql, agentplug-treesitter, rs-search) must close the loop back to gm's own submodule pin as its explicit last step -- push to the submodule's standalone remote alone leaves gm's tracked pointer silently stale, invisible until a git rev-parse HEAD vs origin/main comparison surfaces it. Live-hit twice in one session with two distinct commit pairs before this rule was written. Fix: every such subagent prompt's final step is cd back to gm root, git add <submodule-path>, commit+push gm's own updated pointer in the SAME turn.
