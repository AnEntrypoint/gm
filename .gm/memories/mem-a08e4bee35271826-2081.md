gm-plugkit/bootstrap.js architecture: readShaManifest handles two real formats seen in the wild for plugkit.sha256 -- the checked-in local gm-plugkit/plugkit.sha256 is a JSON manifest ({"plugkit.wasm":"<sha>"}) written by the version-bump automation, while a GitHub-release .sha256 sidecar (fetched directly from plugkit-bin releases) is a standard sha256sum-format line ("<hash>  <filename>"). Tries JSON first since that is this file's own real on-disk shape -- the sha256sum-line regex never matched it, silently returning {} and skipping verification on every bootstrap call before this fix. hasNativeEmbedRunner: plugkit-core's embed.rs probes host_vec_embed before ever loading its wasm-embedded safetensors fallback -- a slim build (feature=slim, no embedded weights) is only safe to fetch on a host that actually answers host_vec_embed for real; agentplug-runner answers it via its shared bert plugin daemon, so the check gates on the runner binary existing under ~/.gm-tools so bootstrap-time artifact selection and runtime embed-delegation eligibility never disagree. Absence means no host_vec_embed answer will ever come, so fetching fat (which carries its own wasm-side embedding fallback) is the only safe choice. downloadFromGithubReleases: a slim fetch that 404s (older release predating slim publish, or asset genuinely missing) falls back to fetching fat rather than failing bootstrap entirely -- a host capable of running slim is also always a valid fat-build host (fat is a strict superset of slim's capability), so the fallback never produces incorrect behavior, only a larger download. pruneOldVersions keep-token bug: verDir uses a '-slim' suffix for the slim cache slot, so the keep-token passed to pruneOldVersions must match that same suffix or it deletes the directory just populated by THIS run before copyWasmToGmTools can read from it -- live-witnessed a real end-to-end bootstrap() run on a native-runner host hit exactly this ('pruned .../v0.1.906-slim' followed by an ENOENT on the immediately-following copy, because 'v0.1.906' != 'v0.1.906-slim').
