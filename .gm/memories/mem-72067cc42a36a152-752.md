PLAN-phase PRD density rule: apply 'every possible' to every noun the request names -- exhaustive task/validation/mutable/corner-case/caveat/failure-mode/interaction/empty-overflow-reentry/degenerate-input enumeration, not minimal coverage. Single-digit PRD on a non-trivial request means the enumeration stopped early. Second-pass transform: feed the first-pass list back, each row's own corner cases become new rows, close only when the transform yields nothing new. High-tens-to-hundreds of rows is the expected shape for a long-horizon request -- row count at PLAN is the only protection against silent residuals surfacing later at COMPLETE. Sparse PRDs orphan work; dense PRDs make completion an observable, checkable state rather than a felt one.
