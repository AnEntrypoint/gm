All plugkit functionality including embedding inference is encapsulated in the rust+wasm runtime. Host-side embedding backends (Node ONNX runtime, worker threads, JS BERT inference) are forbidden. The host_vec_embed import in plugkit-wasm-wrapper.js is a no-op stub that returns -1, which signals the wasm probe to fall back to the in-wasm BertModel. Performance problems in wasm are fixed in rust (smaller model, quantization, simpler load path) — never by outsourcing to the host. Plugkit is self-contained in the wasm binary; the host provides only filesystem, network, and exec shells.
