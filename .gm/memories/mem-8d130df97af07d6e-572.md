gm-method / Workflow tool quirk: the Workflow tool's `args` value arrives inside the workflow script as a JSON-ENCODED STRING in this harness, not as the parsed object the tool docs imply. A script reading `args.surfaces` directly throws (`pipeline() expects an array` / `surfaces not an array; args=string`). Guard every workflow script that takes structured args with: `let a=args; if(typeof a==='string'){try{a=JSON.parse(a)}catch(_){a=null}}` then read fields off `a`. Witnessed: two failed runs (agent_count:0, ~13-53ms) with `args=string`, fixed by JSON.parse guard.
