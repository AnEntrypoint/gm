agentplug-host registry.rs dispatch_on/wasmtime store discipline: dispatch_on is the SINGLE shared path both ProjectPlugins::dispatch and host_plugin_call (cross-plugin dispatch) go through, specifically to avoid wasmtime's 'object used with the wrong store' panic (store/data.rs:213) which fired the first time host_plugin_call tried to drive a sibling Instance using the calling plugin's own Caller instead of the target's own Store. Every dispatch_on call refreshes store.data().set_cwd(caller_root) and set_siblings(caller_siblings) FIRST, before the guest call -- a shared instance (bert/treesitter/libsql/gm) reuses the same HostState across every project's dispatch, so without this refresh, host_cwd/host_fs_*/host_kv_* would resolve against whichever project happened to instantiate the Store first, not the actual caller. Epoch deadline gotcha: wasmtime's default epoch deadline is 0 ('already elapsed'), and a deadline set once at instantiation does NOT auto-refill -- it's a fixed epoch value computed at set-time, so every future epoch-instrumented call on a REUSED Store (SharedPluginPool slots persist across many dispatches) traps immediately once exceeded unless explicitly re-armed. Must re-arm via store.set_epoch_deadline(...) before EVERY guest call on this function's path, including the plugkit_alloc calls (themselves epoch-checked guest exports) -- arming only right before the final dispatch_fn.call left the earlier alloc calls running against the PREVIOUS dispatch's already-possibly-exceeded deadline, live-witnessed as a 'plugkit_alloc call trapped' panic surfacing from write_guest_bytes on a different call site reusing the same Store slot. DISPATCH_CALL_DEADLINE_SECS=40s is deliberately set above both SharedPluginPool::ACQUIRE_TIMEOUT_MS (20s) and host_git's GIT_SUBPROCESS_TIMEOUT_MS (15s) so a call that legitimately needed the full pool-acquire wait still has runway left. Export-name compat: dispatch_on tries 'plugin_call' (the agentplug-native convention) first, falling back to 'dispatch_verb' (gm.wasm's pre-existing export name from plugkit-core's own cascade, predating the agentplug ABI) so both plugin generations dispatch through one code path unchanged.
