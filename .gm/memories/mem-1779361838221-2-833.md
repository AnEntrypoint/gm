thebird (AnEntrypoint/thebird) runs the same plugkit wasm in a real browser, hosts freddie as its agent surface, and serves the libsql stack and embedding stack from that single wasm binary. This is why everything must live IN the wasm: any host-side compute (Node ONNX, JS BERT, child_process git) breaks browser parity. The rust wasm provides 1:1 CLI functionality to thebird users via the same code path the CLI agents use. The host (Node, browser, deno) provides only thin shims: filesystem (or IndexedDB/OPFS in browser), fetch, log, exec_js (Function() in browser, child_process in Node). host_has_git() probes the host; when it returns false the wasm refuses git operations cleanly with HOST_NO_GIT_SENTINEL — browser deployments cannot reach COMPLETE locally and must be driven by an outer Node host that owns the worktree.
