agentplug-runner is the SOLE gm spool loader. The JS wasm-host (gm-plugkit/plugkit-wasm-wrapper.js + gm-plugkit/wrapper/) and the redundant gm-runner native host are retired -- there is no fallback loader. gm-plugkit stays only as the thin launcher edge: bun x gm-plugkit@latest spool -> cli.js::tryDelegateToRunner execs ~/.gm-tools/agentplug-runner and exits; bootstrap.js still seeds per-project wiring + downloads plugkit-slim.wasm (sha256-pinned) from plugkit-bin. bootstrap.js::startSpoolDaemon launches agentplug-runner or returns a loud actionable error naming the runner + 'bun x gm-skill install' when it is absent -- never a silent no-loader state. bin/install.js HARD-requires agentplug-runner: downloads the sha256-verified binary from AnEntrypoint/agentplug-bin for the host platform and main() returns 1 with a FATAL message when no binary is published (agentplugRunnerAssetName()==null) or the download fails. The browser verb is native in agentplug (crates/agentplug-host/src/browser.rs + embedded cdp_eval.js), so browser root-causing edits go to AnEntrypoint/agentplug, not the deleted JS wrapper. ensureWrapperFresh/killStaleWatchers/resolveNodeRuntime and all plugkit-wasm-wrapper references were removed from gm-plugkit.
