SITE-BUILD invocation (gm site, witnessed 2026-06-25): the gh-pages site is a TWO-step build and the steps run in DIFFERENT cwds. Step 1: `npm run build` IN site/ (flatspace aggregate --input platforms.json --output platforms-out.json). Step 2: `npx flatspace@latest build` at REPO ROOT, which reads root flatspace.config.mjs and writes dist/ (paper/index.html etc). Running `flatspace build` in site/ fails with 'no flatspace.config found -> falling back to legacy bun build -> Module not found src/build.js' because the config is at root, not site/. dist/ is gitignored. The deployed SDK is runtime-loaded from unpkg @latest (the site/package.json ^0.0.x pin only feeds flatspace aggregation, never the shipped SDK). To browser-witness a docs/*.html edit: rebuild both steps, then page.evaluate the served /paper/ (or other route) on the local dist server.
