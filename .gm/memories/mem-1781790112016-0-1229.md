LEARNING-OBSERVABILITY invariant: the learning layer EMITS its internals as structured gm-log events so the agent observes them directly; a learning internal that a sniffer can only heuristically guess is a blind-spot to fix by emission. rs-plugkit/src/code_index.rs recall_at emits a 'recall' event {sub:'rs_learn', query, hit, top_score(=1-distance), mode(vector_top_k|fallback_like), n_hits, namespace} via wasm_dispatch::emit_event at each success point; gmsniff --recall-scores/--recall-misses/--recall-modes filter on event=='recall' (sub-agnostic, robust to the wrapper's evt: sub-tagging) to reveal real per-recall behavior. route_hint is surfaced in the instruction response; memorize_fired/memorize_reject/table_dropped are emitted and consumed by gmsniff modes; the long-tail learning feedback (MicroLoRA/FastGRNN/EWC applications, VERIFY quality self-reports) is not yet emitted (emit only when a concrete blind-spot bites). Both gmsniff and ccsniff already expose raw --json event/transcript dumps + --tree/--efficiency timelines + per-session internals + the learning modes, so new reveal-modes are guarded surface-growth -- the lever is making existing blind modes functional via emission, not redundant new modes.
