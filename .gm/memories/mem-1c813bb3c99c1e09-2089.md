agentplug daemon.rs wasm memory + Instant-underflow lessons: (1) WASM linear memory is architecturally monotonic (memory.grow has no inverse in the spec) -- a shared plugin Store (gm/bert/treesitter/libsql) that peaks during one workload retains that peak for the process's whole lifetime; PLUGIN_IDLE_EVICT_MS only removes per-project ProjectPlugins map entries, never touches the SHARED_PLUGINS static's actual Stores. Measured on this host: 350MB fresh daemon, 538MB with all 4 plugins instantiated, 1545MB after one full-repo codeinsight pass, staying there afterward -- VirtualQueryEx attributed the jump to a single ~1285MB contiguous PAGE_READWRITE region, bert's own grown linear memory. Judge memory changes by PrivateMemorySize64, never WorkingSet64: Windows trims a cold working set hard (forced EmptyWorkingSet dropped WorkingSet 1545.8MB->1.0MB while PrivateMemorySize64 held at 1546.6MB), which looks like a real release when nothing was freed. Fix: release_shared_plugin('bert') on a short idle cadence (SHARED_PLUGIN_RELEASE_IDLE_MS=2min) reclaims the Store's pages; treesitter/libsql/gm stay small so releasing them isn't worth the re-instantiation churn except gm which is excluded entirely (holds live orchestrator state). A full self-recycle (process exit, fresh respawn on next real dispatch) reclaims everything after SELF_RECYCLE_IDLE_MS=1hr fully idle. (2) Two separate live-reproduced Instant-subtraction-underflow panics on Windows: `Instant::now() - registry_poll_interval` (to pre-date the timer for an immediate first poll) and `Instant::now() - Duration::from_millis(PLUGIN_IDLE_EVICT_MS)` both panic ('overflow when subtracting duration from instant') on a freshly-booted process (e.g. right after a machine reboot) whose own monotonic-clock epoch is younger than the subtrahend -- crashed the daemon on every boot attempt. Fixed via a first-tick boolean flag (first_registry_poll_pending) instead of pre-dating arithmetic, and `checked_sub(...).unwrap_or_else(Instant::now)` (None correctly means nothing is old enough to evict yet) for the eviction cutoff.
