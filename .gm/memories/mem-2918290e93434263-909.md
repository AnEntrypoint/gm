## Process-of-elimination + manual-real-services-witness paradigm (gm method, EVERYWHERE)

Process-of-elimination is the gm debugging paradigm on every surface (code, wasm, cascade, browser, the spooler itself): enumerate candidate causes as mutables and eliminate each by a witness read against REAL input (exec_js / codesearch / Read / browser page.evaluate), each elimination revealing the next mutable, never guess-and-restart / a-b-test / shotgun-variants. Manual-real-services-witness is the gm verification paradigm EVERYWHERE: every verification is manual labour against the real thing -- the single mock-free test.js, the live page, the real service, the live wasm -- never an automated unit/mock suite standing in for the real-services witness (the conventional-testing tell-tale gm replaces). Both are stated in instructions/execute.md (the served EXECUTE prose) so they reach every in-session LLM.
