Cascade pipeline (gm/rs-* monorepo): push to any rs-* sibling (rs-exec, rs-search, rs-codeinsight, rs-learn) triggers cascade.yml -> gh workflow run invokes rs-plugkit release.yml via PUBLISHER_TOKEN. rs-plugkit cargo-pulls latest sibling crate revs at build time, emits ONE plugkit.wasm (~149MB, embeds bge-small), publishes to plugkit-bin Releases + npm plugkit-wasm, auto-bumps gm.json plugkitVersion + bin/plugkit.wasm.sha256. The version-bump commit triggers publish.yml which npm-publishes gm-skill (repo root) + gm-plugkit (gm-plugkit/) and force-pushes SKILL.md to the AnEntrypoint/gm-skill back-compat mirror. Three npm packages: gm-skill, gm-plugkit, plugkit-wasm. PUBLISHER_TOKEN must be set in rs-exec/rs-codeinsight/rs-search. Never cargo build/update locally - push, let CI build.
