host_exec_js is synchronous (rs-plugkit wasm host mechanism): the wasm host runs the exec_js verb via Node child_process.spawnSync, so a long subprocess BLOCKS the watcher heartbeat for its full duration. Every exec_js dispatch must pass a real positive per-call timeoutMs (missing or zero is a hard host error). Orphaned background work is unsupported under wasm - there is no detached/async exec; spawnSync is the only path. This is the same blocking-heartbeat family as the git_finalize network push that needs busy_until.
