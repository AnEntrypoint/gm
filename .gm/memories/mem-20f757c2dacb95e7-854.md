rs-plugkit state-machine internals (single-crate detail): the PLAN->EXECUTE->EMIT->VERIFY->COMPLETE state machine lives natively in rs-plugkit at rs-plugkit/src/orchestrator/{mod,state,transitions,mutables,memorize}.rs. Plugkit owns phase tracking, mutables resolution, memorize firing, transition legality AS data structures + gate checks, but the agent triggers every operation by dispatching an orchestrator verb over the wasm surface: transition, mutable-resolve, memorize-fire, phase-status, instruction, residual-scan, auto-recall. The gm-skill harness routes the agent verb writes to plugkit; the harness never reimplements the state machine, the agent never expects plugkit to act without a verb. Polling the spool output dir (sleep && ls) instead of reading the response file is the canonical misuse - plugkit is synchronous from the agent view.
