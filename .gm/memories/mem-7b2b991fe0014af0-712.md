wasm_dispatch/verbs.rs history (rs-plugkit): DIAGNOSTIC_EVENT_COOLDOWN_MS pattern -- a plain fire-once AtomicBool latch would permanently silence a diagnostic event after the FIRST occurrence for the lifetime of the shared gm.wasm instance, since this instance is a single process-wide store serving every concurrently-active project (agentplug-host's is_stateless_shared_plugin) -- that would silence the signal for every project after the first hit, forever, until the daemon process restarts. Cooldown timestamps (5 min) re-arm the event so a genuinely recurring condition keeps surfacing periodically instead of going permanently dark. Applied to recall_score_unavailable and discipline_sigil_ignored events.
