gm memory storage mechanics: canonical memory is human-readable md at .gm/memories/<key>.md (project facts) and .gm/disciplines/<name>/memories/<key>.md (transferable discipline knowledge), frontmatter key/ns/created/updated, UTF-8 no BOM, LF (text shares cross-machine/session/CI as reviewable diffs). The md read path is live -- recall serves the md-derived index once a project corpus digest converges. Legacy flat-JSON KV (.gm/disciplines/<ns>/*.json) is a frozen fallback + embedding-reuse source, tracked until a flat-write-removal follow-up retires it. Derived state (embeddings/*-vec, codeinsight manifests/digests, local sqlite/index dbs) is rebuildable-from-corpus, listed one-by-one between managed gitignore markers (parent-re-include caveat). Project-local persistent state always goes to .gm/<name>/, never a top-level dotfile/dotdir.
