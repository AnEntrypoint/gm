Spool dispatch gates (gm lib/spool-dispatch.js): checkDispatchGates(sessionId, operation) implements marker-file gate logic controlling tool use, writes, git ops. Reads marker files .gm/prd.yml (existence triggers needs-gm gate), .gm/mutables.yml (unresolved entry blocks Write/Edit/git), .gm/needs-gm (written by bootstrap, read by dispatcher), .gm/gm-fired-<sessionId> (written by gm skill, cleared at turn start), .gm/residual-check-fired (one-shot residual-scan per stop window). Returns {allowed:bool, reason:string}. Gates checked at CLI/bootstrap layer before tools execute; on denial the reason text surfaces to the model. Gate denials never mutate command args - they surface policy as imperative instruction via reason string.
