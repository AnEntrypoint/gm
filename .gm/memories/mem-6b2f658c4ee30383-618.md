gm-method: rs-plugkit orchestrator::state::Phase is a String newtype, not a fixed Rust enum, because a project can extend the FSM via .gm/instructions/fsm/graph.json to define phases beyond the six built-ins with no Rust rebuild. Always stored/compared uppercase. The six built-ins stay as associated fn constructors purely so call sites read the same as before migrating off the old enum variants. Whether a phase name is LEGAL (has a registered node in the active graph) is the fsm modules job, checked at transition time, never encoded in the Phase type itself. Phase::parse always succeeds on any non-empty string.
