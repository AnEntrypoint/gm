CHROME LIFECYCLE FIX (2026-05-29, gm-plugkit wrapper edit shipping via gm-plugkit npm): user reported agents work around a "known chrome crash" (chrome left open then disconnected) by closing ALL chromes; the working chrome should auto-close on idle and seamlessly re-open on next use. ROOT CAUSE (the load-bearing bug): lastBrowserActivityMs was declared `let` inside runSpoolWatcher but assigned inside makeHostFunctions (a DIFFERENT function scope) at 3 browser-dispatch sites via try{...}catch — the assignments wrote a different/undefined binding and silently failed, so the browser-idle-close timer measured watcher-UPTIME not browser-IDLENESS and effectively never auto-closed correctly. Also the sweep closed ALL chromes in the cwd then unlink-all the ports/sessions files (a baked-in close-all, the same workaround). FIX (4 parts): (1) removed the 3 cross-scope assignments; added module-level stampBrowserLastUse(cwd,sid) that writes ports[sid].lastUse and persists, called at all 3 dispatch sites. (2) added lastUse:Date.now() to fresh + reused ports entries. (3) rewrote the idle sweep to per-session: extracted pure selectIdleBrowserSessions(ports,now,limitMs) to gm-plugkit/browser-idle.js (wrapper requires it), closes only entries idle>=limit, deletes only those sids from ports AND sessions (mirrors collision path, no unlink-all, no global reset), emits per-session browser.idle-closed{sid,pid,idle_ms} (lifecycle event NOT deviation). (4) default idle limit tightened 60min->10min (seamless reopen makes long linger pointless), PLUGKIT_BROWSER_IDLE_LIMIT_MS override kept. Seamless RE-OPEN already existed: getOrCreateBrowserSession startManagedBrowser branch re-opens transparently on next verb after idle-close. ORPHAN/DISCONNECT reclaim already existed: getOrCreateBrowserSession cdpOk check (fetch /json/version) detects disconnected-but-alive chrome and gracefulCloseBrowser(collision:cdp-dead)+respawn. AGENT WORKAROUND REMOVED: lib/browser.js closeAllSessions(opts) now refuses unscoped blast, requires {sessionId} or {all:true} (zero internal callers; was agent-invoked routine only). Test: gm-plugkit/browser-idle.test.js 5 cases all pass, node --check wrapper OK. Live validation of the new idle behavior is cascade-gated (ships via gm-plugkit npm; running watcher is old wrapper until published+booted).
