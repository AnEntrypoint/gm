Residual-triage rule (AGENTS.md amendment 2026-05-23): every agent session must triage every entry in `git status --porcelain` before declaring COMPLETE. Triage = decide (a) commit, (b) add to managed gitignore block, or (c) revert. The label 'pre-existing residual' is the triage decision (a) when work is real, or (c) when stale — NOT a stop excuse. Name-and-stop is admissible only when triage requires external authority. For local-only residuals, the agent triages this turn.
