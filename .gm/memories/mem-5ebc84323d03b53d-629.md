blender-kit and godot-kit lang/ plugin contract integrates with plugkit via the rs-plugkit lang verb (wasm_dispatch.rs fn lang, routed at the dispatch match): spool .gm/exec-spool/in/lang/<N>.txt body {projectDir (absolute), command (e.g. exec:blender / exec:gdscript), code}. The verb resolves projectDir/lang/*.js (excluding loader.js), validates shape {id, exec:{match RegExp, run fn}}, first-match-wins, runs via host_exec_js spawnSync, returns {ok, plugin_id, output, ms}. Both kit plugins try HTTP bridge port 6009 first then fall back to headless engine binary. The SPEC until-verb-lands prose is stale - the verb is live.
