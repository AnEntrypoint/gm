ATOMIC SPOOL BOOT (gm-method, gm-plugkit/cli.js): `bun x gm-plugkit@latest spool` is one atomic blocking call -- startSpoolDaemon() spawns the watcher detached, then waitForWatcherHeartbeat(spoolDir,30000,12000) polls .status.json (sync-sleep via Atomics.wait, no subprocess) and only exit(0)s once ts is fresh (<12s) or busy_until>now; on 30s timeout writeCliError('watcher-heartbeat-timeout')+exit(1). Eliminates the old `& + sleep 8 + re-cat` agent boot dance -- the call returns only when serving. Idempotent: an already-alive fresh watcher returns immediately.
