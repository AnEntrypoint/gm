On a machine running multiple gm-driven project watchers, ~/.gm-tools/ is a SINGLE shared install directory across ALL projects -- every project watcher loads the same wrapper.js and checks its sha against the shared on-disk copy every 10s. A direct node/bun invocation of gm-plugkit cli.js/bootstrap.js run FROM INSIDE the gm source repo checkout itself (bypassing bun x gm-plugkit@latest, which always fetches an isolated npm-temp copy) will copy the repo local dev-tree wrapper.js over that shared install, instantly making every other project on the machine detect wrapper-sha-drift and cascade-kill+respawn simultaneously -- a machine-wide crash-loop storm. Lesson: never test/debug gm-plugkit cli.js by running node/bun directly against the repo checkout on a machine with other live gm watchers -- always use bun x gm-plugkit@latest for live-system testing. Fixed in gm-plugkit/bootstrap.js: ensureWrapperFresh() refuses the shared-install overwrite from a detected gm source-repo checkout unless GM_PLUGKIT_ALLOW_DEV_WRAPPER_OVERWRITE=1 is set.
