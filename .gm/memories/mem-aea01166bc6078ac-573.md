gm-method: rs-plugkit orchestrator::gm_dir() (and its underlying resolve_project_root_with_retry) resolves fresh on EVERY call, never cached via OnceLock, because a single shared gm.wasm instance can serve multiple projects (agentplug-runners daemon architecture) and a cached root would permanently bind that instance to whichever project first resolved it. The git rev-parse call routes through host_git, resolved by the host against callers cwd, correct for whichever project the current dispatch is running under regardless of how many projects share the wasm Instance.
