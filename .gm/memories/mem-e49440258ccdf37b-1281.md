embed.rs architecture (rs-plugkit) embed_texts_batch: batches every input through BERT in ONE model.forward call (batch dimension = texts.len()) instead of one forward pass per text, amortizing candle's vectorized matmul cost over a real batch. Falls back item-by-item (via embed_text, including its plain-text cache) whenever the batch path isn't usable: host-delegated mode (no wasm model loaded), model/tokenizer missing, or any tokenize/tensor-build step failing for the whole batch. try_host_embed is per-item only (host_vec_embed has no batch entry point) -- if the host path is live, it's used per-item for the uncached set rather than falling through to the wasm model, matching embed_text_uncached's own priority order. Sub-batch capping: BERT's attention matrix is O(batch_n * heads * max_len^2) per layer -- padding every item in an unbounded-size batch to the batch's own longest sequence caused a real OOM crash live-witnessed in a session (a ~1.8GB single allocation) once batch_n and max_len were both large at once. Fixed by capping sub-batches by BOTH item count (MAX_SUBBATCH_ITEMS=32) and total padded elements (batch_n * max_len <= 32*512), so no single model.forward call can allocate past a bounded ceiling regardless of how many/how long a file's chunks are.
