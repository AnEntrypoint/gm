Browser tooling uses neutral names in every agent-facing surface: managed browser session, browser runner, headless chromium driver. The internal implementation calls are findBrowserRunner and runBrowserRunner; env vars are GM_BROWSER_RUNNER_PATH and GM_BROWSER_RUNNER_KEY (with back-compat reads of legacy names). Profiles are per-project under <cwd>/.gm/browser-profile/ with a per-pid fallback when the primary SingletonLock is held. The browser verb body is either 'session <subcommand>' (raw passthrough) or an arbitrary JS expression evaluated in the page context; the return envelope is {ok, stdout, stderr, exit_code, session_id?}. No agent-side library imports — every action is a body string dispatched via .gm/exec-spool/in/browser/<N>.txt.
