DEPLOY-KILL PROCEDURE (predictable wrapper-fix deploy, witnessed 0.1.611): to deploy a wrapper/supervisor fix you kill the running watcher so the supervisor respawns it loading fresh on-disk code. To avoid a FALSE supervisor.watcher-exited-unexpectedly severity:critical (the force-kill exits 4294967295), FIRST write .gm/exec-spool/.shutdown-reason.json {reason:external-planned, ts:Date.now()} THEN kill the watcher pid. external-planned is in supervisor.js plannedReasons so the exit logs severity:info, not critical. The mechanism (SHUTDOWN_REASON_PATH read by readShutdownReason in the exit handler) already exists; writing the reason first is the agent-side step. Without it the deploy-kill looks like a crash in gmsniff.
