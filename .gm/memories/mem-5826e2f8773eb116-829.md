Plugkit is rust+wasm only. Every capability — verb dispatch, orchestrator state, embedding inference, vector search, exec, fs, kv, codesearch, memory, browser control — lives inside the wasm binary. The host (Node, browser, bun, deno) provides only minimal shims: filesystem syscalls, network fetch, exec_js, and a few logging hooks. NEVER add a JS/Node/browser-side compute backend (ONNX runtime, embedding worker, JS BERT, host-side tokenizer). The rule is universal because the same wasm runs in AnEntrypoint/thebird in the browser hosting freddie — splitting compute between host and wasm would break browser parity. host_vec_embed and similar host imports stay as no-op stubs that fall back to in-wasm impls. Performance is fixed in rust (model choice, quantization, simpler load) — never by outsourcing to the host.
