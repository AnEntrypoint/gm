bootstrap.js/ensureReady clean-state defect: after `rm ~/.gm-tools/plugkit.{wasm,version}`, `bun x gm-plugkit@latest spool` downloads+extracts the wasm to ~/AppData/Local/plugkit/bin/v<ver>/ but exits without copying it to ~/.gm-tools or writing plugkit.version, so the wrapper self-heals 'wasm not installed' and no watcher boots. Manual recovery: node copy cache wasm -> ~/.gm-tools/plugkit.wasm + write version file, then `node ~/.gm-tools/plugkit-wasm-wrapper.js spool` boots fine. Fix belongs in the gm-plugkit ensureReady clean-state path (copy + version-write ordering / early-exit). Reachable only with a gm-plugkit edit + cascade publish.
