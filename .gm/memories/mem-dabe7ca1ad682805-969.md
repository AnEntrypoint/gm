search-discipline sharpened (2026-05-28 audit): codesearch indexes ONLY the conversation cwd (gm), so the rule splits cleanly. (1) IN-CWD source-symbol lookup (a function/symbol/definition in gm's own tree) MUST be a codesearch dispatch, never Grep/bash-grep - this is the real violation (caught 2 true positives this fire: Grep for getOrCreateBrowserSession + session-routing on gm-plugkit/wrapper, both gm-indexed). (2) CROSS-REPO search (rs-plugkit/rs-exec/ccsniff/etc, a sibling outside cwd) has NO codesearch index, so bash-grep/cd-grep there is unavoidable and NOT a deviation - ccsniff b724dc5 now exempts it via targetsOutsideCwd. (3) config-value greps (gm.json plugkitVersion) + build-output greps (dist/, gitignored) are acceptable bash, not source-symbol search. The recurring self-deviation (this fire + iter121) was conflating all three; the detector + this rule separate them so future fires neither violate (in-cwd) nor over-correct (cross-repo/config).
